# 企业BBS系统 — 需求分析

## 技术栈

| 层级 | 技术选型 | 版本/说明 |
|------|---------|----------|
| 后端语言 | Java | 21 (LTS) |
| 后端框架 | Spring Boot | 3.x |
| 数据库 | MySQL 8.x（本地开发）/ PostgreSQL（部署） | |

| 构建工具 | Maven | |

## 用户体系

### 角色定义

| 角色 | 说明 | 权限范围 |
|------|------|---------|
| 普通用户 | 注册用户 | 发帖、删自己的帖、评论、修改昵称、删除自己的账户、浏览所有内容 |
| 管理员 | 系统启动自动创建预设账号 | 普通用户所有权限 + 板块CRUD、删任何人的帖子 |

### 注册字段

- 账号（唯一，仅限字母和数字，注册时查重）
- 密码（无强度要求，BCrypt加密，不可修改）
- 昵称（可随时修改，无次数限制，顶部导航栏点击修改）

### 账号生命周期

| 阶段 | 说明 |
|------|------|
| 注册 | 账号 + 密码 + 昵称，注册成功后自动登录跳转 |
| 登录 | 账号密码登录，成功后返回JWT Token（有效期2小时），Token存localStorage |
| 使用 | 携带Token访问各接口 |
| 注销（退出登录） | 仅退出当前登录状态，Token加入内存黑名单，可重新登录。无需二次确认 |
| 删除账户 | 永久删除，需二次确认弹窗。硬删除用户+级联硬删除帖子，评论保留显示"已注销用户" |
| 找回密码 | 不做 |

### 删除账户后数据处理

| 数据类型 | 处理方式 | 原因 |
|---------|---------|------|
| 用户发布的帖子 | 级联硬删除 | 保持数据整洁 |
| 用户发布的评论 | 保留，显示"已注销用户"标记 | 避免评论断裂 |
| 用户账号 | 硬删除 | 用户明确要求永久删除 |

## 板块管理（仅管理员）

### 功能

| 功能 | 说明 |
|------|------|
| 创建板块 | 设置名称、描述 |
| 修改板块 | 修改名称、描述 |
| 删除板块 | 级联删除该板块下所有帖子，需二次确认弹窗 |
| 查看板块列表 | 所有用户可见（左侧边栏） |

### 板块属性

| 属性 | 说明 |
|------|------|
| 名称 | 唯一，不可重复 |
| 描述 | 可选，板块简介 |

## 帖子管理

### 帖子生命周期

| 阶段 | 说明 |
|------|------|
| 创建 | 填写标题、内容（纯文本，无格式）、选择板块 |
| 发布 | 点击发布立即上线，无草稿保存 |
| 审核 | 无审核机制，发布即可见 |
| 修改 | 发布后不可修改 |
| 删除 | 普通用户删自己的，管理员可删任何人的，硬删除，点击直接删除无需确认 |

### 帖子内容

| 属性 | 说明 |
|------|------|
| 标题 | 必填，无长度限制 |
| 内容 | 纯文本，支持换行，无格式，无长度限制，不支持图片 |
| 摘要 | 后端自动生成，截取正文前100个字符 |
| 发布者 | 显示用户名 |
| 发布时间 | 精确到秒 |

### 帖子列表页展示

| 项目 | 说明 |
|------|------|
| 展示内容 | 标题 + 摘要 + 作者 + 发布时间 |
| 排序 | 按发布时间倒序（最新在前） |
| 分页 | "加载更多"按钮，每次加载20条 |
| 默认视图 | 展示所有帖子（全站最新） |
| 板块筛选 | 左侧边栏点击板块筛选，顶部有"全部"选项 |

### 帖子详情页结构（从上到下）

标题 → 作者 → 发布时间 → 内容 → [删除按钮(作者/管理员)] → 返回按钮 → 评论列表 → 评论输入框

## 评论管理

### 评论规则

| 项目 | 说明 |
|------|------|
| 层级 | 仅一级评论，无楼中楼 |
| 删除 | 评论一旦发表不可删除 |
| 展示 | 仅显示内容，不显示作者昵称和时间 |
| 已注销用户 | 评论者删除账户后，评论显示"已注销用户"标记 |

### 评论展示

| 场景 | 展示方式 |
|------|---------|
| 帖子详情页 | 展示全部评论（不分页） |
| 评论输入 | 单行输入框（input），帖子内容下方 |

## 认证与授权

### 登录认证

| 项目 | 方案 |
|------|------|
| 认证方式 | JWT |
| Token载体 | HTTP Header Authorization: Bearer token |
| Access Token有效期 | 2小时 |
| Refresh Token | 不做 |
| Token黑名单 | 内存实现（ConcurrentHashMap），注销和删除账户时加入 |
| Token存储 | 前端localStorage |

### 权限控制

| 接口类型 | 访问权限 |
|---------|---------|
| 公开接口（登录、注册） | 无需Token |
| 所有其他接口 | 必须登录，需Token |
| 游客 | 不能浏览任何内容，必须登录 |

## 数据库设计

### 表结构

| 表名 | 说明 | 核心字段 |
|------|------|---------|
| users | 用户表 | id, account, password, nickname, role, is_deleted, created_at |
| boards | 板块表 | id, name, description, created_at |
| posts | 帖子表 | id, title, content, summary, user_id, board_id, created_at |
| comments | 评论表 | id, content, post_id, user_id, created_at |

### 设计决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 表名 | 复数形式（users, boards, posts, comments） | 规范命名 |
| 字段命名 | 下划线风格（user_id, created_at） | 数据库惯例 |
| 主键 | 自增BIGINT | 简单高效 |
| 外键 | 应用层控制，不使用数据库外键 | 更灵活 |
| 用户删除 | is_deleted软标记 | 保留评论外键引用 |
| 帖子删除 | 硬删除 | 保持数据整洁 |
| 评论删除 | 硬删除（仅评论不可删除） | 一级评论无子评论 |
| 摘要生成 | 后端截取前100字符 | 省带宽 |

## 接口规划

### 认证模块

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/register | 注册（返回token，自动登录） |
| POST | /api/auth/login | 登录 |
| POST | /api/auth/logout | 注销（退出登录，Token加入黑名单） |

### 用户模块

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/user/me | 获取当前用户信息 |
| PUT | /api/user/nickname | 修改昵称 |
| DELETE | /api/user/me | 删除账户（硬删除+级联） |

### 板块模块

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/boards | 创建板块（管理员） |
| PUT | /api/boards/{id} | 修改板块（管理员） |
| DELETE | /api/boards/{id} | 删除板块（管理员，级联删帖） |
| GET | /api/boards | 获取板块列表 |

### 帖子模块

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/posts | 发帖 |
| GET | /api/posts | 帖子列表（分页，支持boardId筛选） |
| GET | /api/posts/{id} | 帖子详情 |
| DELETE | /api/posts/{id} | 删帖（作者或管理员） |

### 评论模块

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/posts/{id}/comments | 发表评论 |
| GET | /api/posts/{id}/comments | 获取评论列表 |

### 统一响应格式

```json
{ "code": 200, "message": "success", "data": {} }
```

## 前端设计

### 页面布局

- 顶部导航栏 + 左侧边栏（板块列表）+ 中间内容区
- 仅桌面端，不做移动端适配

### 导航栏（从左到右）

系统名称"企业BBS系统" | [板块管理(仅管理员可见)] | 发帖 | 昵称(可点击修改) | 注销 | 删除账户

### 页面列表

| 页面 | 说明 |
|------|------|
| 登录/注册页 | 同一页面内切换标签，错误提示在字段下方 |
| 主页面 | 左侧边栏+帖子列表/帖子详情/发帖/板块管理 |
| 帖子列表 | 标题+摘要+作者+时间，"加载更多"按钮 |
| 帖子详情 | 标题→作者→时间→内容→删除→返回→评论→输入框 |
| 发帖页 | 标题input+内容textarea+板块下拉+发布/取消按钮 |
| 板块管理 | 当前页面变为管理视图，页面内展开表单增删改 |

### 前端技术

| 项目 | 选型 |
|------|------|
| 框架 | Vue 3 (Composition API) |
| 构建 | Vite |
| UI库 | Ant Design Vue |
| 状态管理 | Pinia |
| HTTP | Axios（统一拦截器） |
| 路由 | Vue Router（hash模式） |
| 路由守卫 | beforeEach判断登录状态 |
| 导航方式 | 前端状态切换，URL不变（仅/login和/main两个路由） |

### 交互细节

| 场景 | 行为 |
|------|------|
| 注册成功 | 自动登录跳转到主页 |
| 发帖成功 | 跳转到帖子列表 |
| 发表评论 | 立即显示（前端先添加） |
| 删除帖子 | 直接删除，无需确认 |
| 删除板块 | 需二次确认弹窗 |
| 删除账户 | 需二次确认弹窗 |
| 注销 | 直接退出到登录页，无需确认 |
| 修改昵称 | 顶部导航栏点击，inline edit |
| 加载状态 | 不显示加载提示，直接呈现结果或空白 |
| 错误提示 | 页面内红色文字 |
| 表单验证 | 提交时统一验证 |
| 空状态 | 空白页面 |

## 后端设计

### 项目结构

- 分层包结构：com.bbs.controller / service / mapper / entity / config / common / util / interceptor
- 全局异常处理（@ControllerAdvice）
- 全局CORS配置
- JWT拦截器
- 内存Token黑名单（ConcurrentHashMap + 定时清理）

### 部署架构

```
客户端(浏览器) → Vercel(前端静态资源) → Render(Spring Boot后端) → PostgreSQL
```
